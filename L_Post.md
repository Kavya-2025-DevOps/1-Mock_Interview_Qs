🚀 #Infosys DevOps Interview 1st Round Interview Questions 🎉

Here are some of the scenario-based and troubleshooting questions that were asked.

An application hosted behind an ALB is returning 503 errors. How would you troubleshoot the issue?

Users are reporting slow application performance. Which AWS services and metrics would you check first?

An Auto Scaling Group is not launching new instances during traffic spikes. How would you investigate?

An RDS database becomes unavailable during business hours. What would be your recovery approach?

How would you perform a zero-downtime deployment in AWS?

The Jenkins master is running out of disk space. What actions would you take?

Build artifacts are not getting uploaded to the artifact repository. How would you investigate?

A Docker container keeps restarting in production. How would you identify the root cause?

Docker image size has grown from 300MB to 3GB. How would you optimize it?

A container works perfectly on a developer machine but fails in Kubernetes. What would you check?

How would you recover if the Docker daemon becomes unavailable on a production server?

A pod is stuck in CrashLoopBackOff. How would you troubleshoot it?

A deployment rollout is stuck and new pods are not becoming ready. What steps would you follow?

Users cannot access an application exposed via Ingress. How would you debug the issue?

One worker node becomes NotReady. What actions would you take?

etcd becomes unavailable in a production cluster. What is your recovery strategy?

Pods are getting evicted frequently due to resource pressure. How would you prevent this?

A service is running but application requests are timing out. How would you troubleshoot networking issues?

Terraform state file is accidentally deleted. How would you recover?

A Terraform apply fails midway after creating some resources. What would be your next steps?

Multiple engineers are modifying infrastructure simultaneously. How would you handle state locking?

A resource was manually changed in AWS but Terraform is showing drift. How would you resolve it?

How would you safely deploy infrastructure changes to production using Terraform?

A production deployment caused downtime. How would you perform rollback and identify the root cause?

Monitoring alerts show increased response time but infrastructure appears healthy. How would you investigate?

Disk utilization on production servers reaches 95%. What immediate and long-term actions would you take?

Kubernetes pods are healthy but users are receiving errors. How would you troubleshoot the complete request flow?

CI/CD pipelines are successful but application changes are not visible in production. What would you check?

A sudden traffic spike causes application degradation. How would you stabilize the platform?
